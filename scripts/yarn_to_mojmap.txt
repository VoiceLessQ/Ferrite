# Yarn -> Mojmap translation table for Ferrite 1.21.11 -> 26.1.2 port.
# Three sections:
#   [PACKAGES]   plain-string replacement on package qualifiers (works on imports
#                AND inline fully-qualified references like `net.minecraft.X.Y`)
#   [CLASSES]    word-boundary replacement (\b...\b) for renamed classes
#   [POST]       plain-string replacement, runs AFTER [CLASSES] - used to
#                undo class-rename clobbers (e.g. java.util.Random getting
#                clobbered by the broad Random -> RandomSource rule)
# scripts/apply_mappings.sh consumes this file.
#
# Ordering rules:
#   - Packages:  more-specific prefixes BEFORE broader ones (longer-first wins).
#   - Classes:   when one rule's output token is another rule's input token (e.g.
#                `Registries -> BuiltInRegistries` and `RegistryKeys -> Registries`),
#                run the consumer FIRST so the producer's output isn't re-eaten.
#   - World vs world.X: NO broad `net.minecraft.world.` rule because its target
#                `net.minecraft.world.level.` starts with the source string and
#                would re-fire on the output of every more-specific world.X rule.
#                The seven net.minecraft.world.<DirectClass> imports are listed
#                explicitly instead.
#
# IMPORTANT NOTE on 26.1.x mojmap quirks (verified against OreVein):
#   - Identifier kept its name (NOT renamed to ResourceLocation), only moved
#     packages: net.minecraft.util.Identifier -> net.minecraft.resources.Identifier.
#   - ItemEntity moved to net.minecraft.world.entity.item (not just world.entity).
#   - BlockState in net.minecraft.world.level.block.state (extra .state. segment).
#   - Registry in net.minecraft.core (NOT core.registries).  But Registries and
#     BuiltInRegistries ARE in core.registries.

# === Phase B method renames (receiver-unambiguous) ===
# Pattern `.OLD(` matches method-call sites only.  Each entry is safe because:
#   - the Yarn method name doesn't collide with any common Java/library method;
#   - the mojmap method name is correct on the same receiver;
#   - or, in Ferrite's source, the only callers are MC types.
# Method names that DO collide (.size, .up, .down, .get, .is, .with, .getBlockPos)
# are NOT here - they need per-file edits.
# IMPORTANT: dots are escaped (\.) so sed treats them as literal `.`, not
# regex any-char.  Without escaping, `.up()` would match `Setup()` as
# `Set` + `.up()` (with `.` matching `t`), corrupting unrelated identifiers.

\.toImmutable()|.immutable()
\.getDefaultState()|.defaultBlockState()
